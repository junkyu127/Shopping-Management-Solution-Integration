🛒 쇼핑몰 통합 관리 솔루션 API 연동 및 유지보수
2년 동안 쇼핑몰 통합 관리 솔루션 **'이지오토'**의 백엔드 개발 및 유지보수를 담당하며, 다양한 이커머스 플랫폼의 데이터를 통합 관리하는 업무를 수행했습니다.

🛠 사용 기술
Language: PHP 7.4 / 8.1, C# (.NET Framework 4.8)

Database: MySQL

Version Control: Git

Communication: REST API (JSON/XML)

🚀 주요 업무 및 성과
1. 쇼핑몰 Open API 연동 및 풀 사이클(Full-Cycle) 관리
국내외 주요 쇼핑몰(네이버 스마트스토어, 쿠팡 등)의 API를 활용하여 이커머스 핵심 운영 프로세스 자동화.

주문 관리: 실시간 주문 확인 및 자동 주문 수집 로직 구현.

배송 및 송장 전송 자동화 로직 설계:

단계별 검증 프로세스: API 실시간 주문 상태 조회 → 내부 DB 배송 데이터 매칭 → 다중 분기 처리를 통한 유효성 검증(Validation) → 일괄/개별 송장 전송 API 호출.

이를 통해 오발송으로 인한 쇼핑몰 패널티 발생을 방지하고 업무 효율성을 극대화함.

CS 및 취소 연동: 쇼핑몰 측의 주문 취소 상태를 실시간 동기화하고, 게시판 API 연동을 통해 통합 고객 응대 시스템 구축.

데이터 정규화(Normalization): 각 플랫폼별 상이한 데이터 규격을 솔루션 내부 표준 포맷으로 가공하는 파이프라인 관리.

2. 업체별 맞춤형 비즈니스 로직(Custom Logic) 관리
특정 업체의 개별 요구사항에 대응하기 위한 전략적 분기 처리 로직 구현.

커스텀 요청 시 시스템 전체의 안정성을 해치지 않도록 코드 독립성을 확보하고, 유지보수를 고려한 로직 분리 수행.

3. 데이터 정합성 유지 및 트러블슈팅
API 안정성: 호출 제한(Rate Limit) 및 타임아웃 발생 시 재시도(Retry) 로직을 통해 데이터 유실 방지.

DB 최적화: 대량의 주문 데이터 조회 시 인덱스(Index) 최적화 및 실행 계획 분석을 통해 리스트 렌더링 및 프로그램 연동 속도 개선.

💡 Troubleshooting & Problem Solving
실무 중 발생한 기술적 난제들을 데이터 분석과 로직 설계를 통해 해결한 경험입니다.

1. 주문 상세 데이터의 독립성 확보를 통한 '개별 배송' 구현
문제: 한 주문 번호 내 복수 상품 존재 시, 특정 상품만 먼저 발송해야 하는 '개별 배송' 요구사항이 있었으나 기존 시스템은 주문 번호 단위로만 처리됨.

해결:

각 쇼핑몰 API 응답 데이터 중 상품별 독립성을 보장하는 키값(주문상세번호, 관리번호 등)을 전수 조사.

해당 독립 키값을 수집 단계에서부터 DB에 저장하도록 스키마를 보완하고, 송장 전송 API 호출 파라미터에 유동적으로 포함시키는 로직 구현.

성과: 상품 단위의 정밀한 배송 관리가 가능해져 업체별 다양한 배송 시나리오를 수용함.

2. 신규 API 연동 및 특화 로직 검증 프로세스 최적화
문제: 쇼핑몰마다 상이한 API 규격으로 인해 운영 환경 적용 시 예측하지 못한 런타임 에러 발생 위험 존재.

해결:

특정 업체를 **테스트 베드(Test-bed)**로 지정하여 실제 주문 데이터를 기반으로 API 호출 규격을 사전 검증하는 프로세스 정립.

하드코딩된 테스트 로직이 검증 완료 후 운영 환경으로 안전하게 전환될 수 있도록 코드 구조 개선 및 분기 처리 레이어 격리.

성과: 신규 연동 시 발생하는 리스크를 최소화하고 유지보수 효율성을 제고함.

📝 업무 경험 요약
사내 보안 정책상 실무 코드를 직접 업로드할 수 없으나, 본 레포지토리는 해당 기간 동안 경험한 기술적 고민과 해결 과정을 기록하기 위해 생성되었습니다.

이커머스 도메인에 대한 높은 이해도를 바탕으로, API 연동 중 발생하는 예외 상황 처리 및 데이터 정합성 확보에 강점이 있습니다.
