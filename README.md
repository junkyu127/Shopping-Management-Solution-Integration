# 🛒 쇼핑몰 통합 관리 솔루션 API 연동 및 유지보수

2년 동안 쇼핑몰 통합 관리 솔루션 '이지오토'의 백엔드 개발 및 유지보수를 담당하며, 다양한 이커머스 플랫폼의 데이터를 통합 관리하는 업무를 수행했습니다.

---

## 🛠 사용 기술
* **Language:** PHP 7.4 / 8.1, C# (.NET Framework 4.8)
* **Database:** MySQL
* **Version Control:** Git
* **Communication:** REST API (JSON/XML)

---

## 🚀 주요 업무 및 성과

### 1. 쇼핑몰 Open API 연동 및 풀 사이클(Full-Cycle) 관리
* **이커머스 운영 자동화**: 국내외 주요 쇼핑몰(네이버 스마트스토어, 쿠팡 등)의 API를 활용하여 핵심 프로세스 구축.
* **주문 관리**: 실시간 주문 확인 및 자동 주문 수집 로직 구현.
* **배송 및 송장 전송 자동화**:
    * **단계별 검증**: API 실시간 주문 상태 조회 → 내부 DB 배송 데이터 매칭 → 다중 분기 유효성 검증(Validation) → 일괄/개별 송장 전송 API 호출.
    * **패널티 방지**: 오발송 위험을 사전 차단하여 쇼핑몰 계정 점수 보호 및 업무 효율성 극대화.
* **CS 및 취소 연동**: 쇼핑몰 취소 상태 실시간 동기화 및 게시판 API 연동을 통한 통합 고객 응대 시스템 구축.
* **데이터 정규화**: 플랫폼별 상이한 데이터를 솔루션 표준 포맷으로 가공하는 파이프라인 관리.

### 2. 업체별 맞춤형 비즈니스 로직(Custom Logic) 관리
* **전략적 분기 처리**: 특정 업체의 개별 요구사항 대응을 위한 로직 분리 및 독립성 확보.
* **유지보수성 향상**: 공통 시스템 안정성을 해치지 않는 선에서 업체별 커스텀 로직 반영.

### 3. 데이터 정합성 유지 및 트러블슈팅
* **API 안정성**: 호출 제한(Rate Limit) 및 타임아웃 발생 시 재시도(Retry) 로직으로 데이터 유실 방지.
* **DB 최적화**: 대량 주문 데이터 조회 시 **인덱스(Index) 최적화 및 실행 계획 분석**을 통해 리스트 렌더링 속도 개선.

## 💡 Troubleshooting & Problem Solving

### 1. 주문 상세 데이터의 독립성 확보를 통한 '개별 배송' 구현
* **문제**: 한 주문 번호 내 복수 상품 존재 시, 특정 상품만 먼저 발송해야 하는 '개별 배송' 요구사항 대응 한계.
* **해결**: 
    * 상품별 독립성을 보장하는 키값(주문상세번호, 관리번호 등) 전수 조사 및 DB 스키마 보완.
    * 송장 전송 API 호출 파라미터에 해당 독립 키값을 유동적으로 매핑하는 로직 구현.
* **성과**: 상품 단위의 정밀 배송 관리가 가능해져 업체별 다양한 시나리오 완벽 수용.

### 2. 신규 API 연동 및 특화 로직 검증 프로세스 최적화
* **문제**: 쇼핑몰마다 상이한 API 규격으로 인한 운영 환경 배포 리스크 존재.
* **해결**: 
    * 특정 업체를 테스트 베드(Test-bed)로 지정하여 실제 데이터 기반 API 호출 규격 사전 검증.
    * 검증 완료 후 운영 환경으로 안전하게 전환할 수 있는 분기 처리 레이어 격리 구조 설계.
* **성과**: 신규 연동 시 발생하는 장애 리스크 최소화 및 유지보수 효율 제고.
